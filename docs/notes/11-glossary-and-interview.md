# 11. Glossary and Interview Answers

## Beginner Glossary

### API

A defined way for software parts to communicate.

### ASP.NET Core

The .NET framework that hosts PayPortal as a web application.

### Assembly

Compiled .NET output such as a `.dll`.

### Authentication

Proving who a user is.

### Authorization

Checking what a user may access or do.

### Blazor

A .NET UI framework that combines Razor markup and C#.

### Class

A definition used to create objects.

### Clean Architecture

A way to separate business code from UI, database, and external technologies.

### Component

A reusable Blazor UI unit stored in a `.razor` file.

### Configuration

Values that control application behavior without changing C# source.

### Dependency Injection

ASP.NET Core creates and supplies the services a class needs.

### Docker

A tool that runs packaged software environments. PayPortal uses it for local
MySQL.

### DTO or Model

A structured object used to move or validate data.

### Entity

A business object that normally has an ID and is stored in the database.

### Entity Framework Core

The ORM that maps C# entities to MySQL tables.

### Foreign Key

A database value that connects one table record to another.

### Identity

ASP.NET Core's system for users, passwords, cookies, roles, and reset tokens.

### Interface

A contract describing operations without selecting an implementation.

### Middleware

A step in ASP.NET Core's HTTP request-processing chain.

### Migration

A versioned database schema change generated by EF Core.

### MySQL

The relational database used by PayPortal.

### NuGet

The package manager for .NET libraries.

### ORM

Object-Relational Mapper. It connects objects in code to relational tables.

### Razor

Syntax that mixes HTML and C# using `@`.

### Repository

A data-access abstraction used to load and save entities.

### Role

A named permission group, such as Merchant or Admin.

### Runtime

The software that executes compiled code. PayPortal uses the .NET runtime.

### Service

A class responsible for an application operation or technical capability.

### SignalR

The live connection technology used by Blazor Interactive Server.

### Solution

A group of related .NET projects, represented by `.sln`.

### User Secrets

Development-only configuration stored outside the repository.

## Common Interview Questions

### What is the difference between C#, .NET, ASP.NET Core, and Blazor?

> C# is the programming language. .NET is the runtime and platform. ASP.NET Core
> is the web framework on .NET. Blazor is the ASP.NET Core UI framework used to
> build interactive pages with Razor and C#.

### Why use Clean Architecture?

> It separates business models and application contracts from technical details
> such as MySQL, EF Core, Identity, file storage, and Blazor. That reduces
> coupling and makes implementations easier to replace or test.

### Why use EF Core?

> EF Core maps C# entities to MySQL tables, supports LINQ queries, tracks
> changes, and versions the schema through migrations.

### Why use User Secrets?

> They keep developer-specific credentials outside Git and are automatically
> loaded by ASP.NET Core in Development. They are not used as the production
> secret store.

### How is production configuration different?

> Production would override configuration using environment variables or a
> managed secret store and use a production MySQL connection, private object
> storage, email service, monitoring, and stronger administrator security.

### How is authorization enforced?

> PayPortal uses role-protected routes and service-level checks. Merchant access
> is also scoped by ownership so one merchant cannot load another merchant's
> data.

### Why Blazor Interactive Server?

> It keeps UI logic in C#, avoids exposing database access to the browser, and
> sends UI updates over SignalR. The tradeoff is that each interactive session
> needs a live server connection.

### How do API credentials stay secure?

> They are generated with cryptographic randomness, the secret is hashed before
> storage, plaintext is shown only when issued or rotated, and old credentials
> are revoked during rotation.

## A Two-Minute Project Explanation

> PayPortal is a merchant onboarding portal built with C# and .NET 8. ASP.NET
> Core hosts the web application, and Blazor Interactive Server provides the UI.
> The solution follows Clean Architecture with separate Domain, Application,
> Infrastructure, and Web projects.
>
> Merchants register, maintain a business profile, upload private KYC documents,
> track reviews, and generate API credentials after approval. Administrators can
> search applications and approve, reject, or request more documents.
>
> ASP.NET Core Identity manages users, hashed passwords, login cookies, and the
> Merchant and Admin roles. EF Core maps domain entities to a MySQL database and
> applies Code First migrations. Local MySQL runs through Docker.
>
> Shared configuration uses appsettings files, development credentials use .NET
> User Secrets outside Git, and production can override settings with
> environment variables or a managed secret store. Authorization is enforced at
> both the route and service levels, including merchant ownership checks.
