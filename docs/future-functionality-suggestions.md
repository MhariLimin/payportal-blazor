# Future Functionality and Page Suggestions

These are additional relevant pages and workflows that would make PayPortal
feel more like a production merchant onboarding system.

## Relevant Pages

1. **Risk Rules:** implemented as a demo page with scoring rules for industry,
   business type, and rejected document count.
2. **Reviewer Assignments:** assign merchants to specific admins and show each
   reviewer their queue.
3. **Document Request Center:** implemented as `Document Requests` for missing
   and rejected evidence tasks.
4. **Merchant Timeline:** superseded by the focused merchant `Activity` page in
   Feedback 5 to avoid duplicate activity views.
5. **API Usage:** implemented with demo credential telemetry.
6. **Admin Settings:** implemented with document, file, password, and SLA
   policy summaries.
7. **Reports:** implemented with conversion, approval, rejection, review time,
   and document completeness summaries.

## Functionality Improvements

1. **Individual document request workflow:** implemented through rejected
   document tasks and resubmission guidance.
2. **Review SLA indicators:** flag applications that have waited too long in
   Pending or Under Review.
3. **Risk scoring automation:** implemented for profile updates and document
   rejection review.
4. **Email notifications:** send account, document, request, approval, and
   rejection emails using a background queue.
5. **Admin notes:** implemented as internal notes on Applications, hidden from
   merchant KYC timelines.
6. **Bulk review actions:** implemented as multi-select export preparation on
   Applications.
7. **Production file scanning:** represented by scan-readiness indicators in
   the review workflow; external scanning integration remains future work.
