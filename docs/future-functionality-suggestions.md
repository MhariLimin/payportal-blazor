# Future Functionality and Page Suggestions

These are additional relevant pages and workflows that would make PayPortal
feel more like a production merchant onboarding system.

## Relevant Pages

1. **Risk Rules:** let admins configure risk scoring rules by country, industry,
   business type, document rejection count, and revenue range.
2. **Reviewer Assignments:** assign merchants to specific admins and show each
   reviewer their queue.
3. **Document Request Center:** let admins request a specific missing document
   type and let merchants respond directly to that request.
4. **Merchant Timeline:** provide one chronological page for registration,
   profile edits, KYC uploads, document reviews, compliance review, approval,
   and API credential events.
5. **API Usage:** show credential last-used time, request counts, error rates,
   and webhook delivery results.
6. **Admin Settings:** configure allowed file types, document requirements,
   password policy notes, and review SLA targets.
7. **Reports:** export onboarding conversion, approval rate, rejection reasons,
   average review time, and document completeness.

## Functionality Improvements

1. **Individual document request workflow:** connect rejected documents to a
   merchant task with a clear resubmission state.
2. **Review SLA indicators:** flag applications that have waited too long in
   Pending or Under Review.
3. **Risk scoring automation:** calculate Low, Medium, or High risk instead of
   leaving it static.
4. **Email notifications:** send account, document, request, approval, and
   rejection emails using a background queue.
5. **Admin notes:** allow internal-only notes separate from merchant-visible
   reviewer notes.
6. **Bulk review actions:** allow admins to assign, tag, or export selected
   applications.
7. **Production file scanning:** scan uploaded files before admins can download
   or verify them.
