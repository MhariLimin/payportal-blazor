# Post-Implementation Feedback 2

This document maps the second feedback pass to the implemented behavior.

1. **Typography:** the UI now uses Inter for body text and Sora for headings.
2. **Density:** shared spacing, card padding, buttons, inputs, side navigation,
   and dashboard metrics were reduced so more content fits on screen.
3. **Merchant dashboard:** Merchant users now see onboarding progress, next
   steps, recent documents, account readiness, and merchant-scoped activity.
4. **Admin pages:** Applications is now positioned as the searchable merchant
   directory. Admin Review was renamed to Compliance Queue and is focused on
   evidence checks and final review decisions.
5. **Company profile milestone:** new merchant registrations start with this
   milestone pending. It completes when the merchant saves the editable profile.
6. **Compliance review milestone:** the UI explains that required document
   upload comes first. Compliance Review starts after required documents are
   present and completes only after an administrator approves or rejects.
7. **Document requirements:** KYC requirements now live in the main document
   card, with uploaded files grouped under the matching requirement.
8. **Feature suggestions:** see `docs/feature-suggestions.md`.
9. **Admin charts:** the admin dashboard now includes funnel bars, decision mix,
   risk attention, and pending-approval workload cards.
