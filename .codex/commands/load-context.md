# Load Codex Context

Use this procedure for `load codex context`, related reload prompts, and
numbered roadmap commands.

1. Read `.codex/CODEX.md`.
2. Read the four always-load files in `.codex/context/`.
3. For planning or feature work, read `roadmap.md` and `specs/index.md`.
4. Resolve numbered work from the roadmap and read only the matching spec.
5. Inspect the current worktree before editing and preserve unrelated changes.
6. Verify with the narrowest relevant command, expanding to `npm run
   typecheck`, `npm run lint`, and `npm run build` as needed.

After loading, briefly name the `.codex` files used, then continue.
